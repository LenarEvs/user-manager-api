# Prompts from this dialog

## 1) Промпт
```text
Задача:
Создай минимальный проект для управления пользователями

Требования
- Зафиксируй зависимости (requirements.txt или pyproject.toml).
```

## 2) Промпт
```text
Задача:
Нужно реализовать в текущем репозитории LangChain агент

Требования:
- Для реализации используется локальный Ollama
- Агент должен храниться в подпапке langchain-agent
- Секреты и ключи не должны комититься
- В репозитории должен быть .env.example
- Необходимо зафиксировать зависомсти (requirements.txt или pyproject.toml).
```

## 3) Промпт
```text
Задача:
Реализуйте tool для вызова API

Требования:
- Tool должен быть оформлен как LangChain tool.
- Внутри tool — явный HTTP-вызов или функция-обёртка.
- Tool возвращает структурированный результат (JSON / dict → string).
```

## 4) Промпт
```text
Задача:
Собрать агента

Требования:
- Настройте системный prompt (роль агента (API-оператор);
ограничения (что он может и не может делать);
правила вызова tool.)
- Подключить API-tool.
- Реализуйте запуск из CLI, например:
python main.py "создай пользователя с именем Alex"
```

## 5) Промпт
```text
Задача:
Определить контракт ответа

Требования:
Ответ агента должен быть в фиксированном формате, например:
Status: success | error
Action: <описание действия>
Data: <результат API>
Errors: <если есть>
```

## 6) Промпт
```text
Задача:
Проверь что все работает корректно

Требования:
- Создай в корне папку report и в ней файл tests.md
- Выполните минимум 5 разных пользовательских запросов.
- Зафиксируйте запросы и ответы в tests.md
```

## 7) Промпт
```text
все проверки прошли с ошибками, исправь их и когда будет успешно выполни повторно тест и зафиксируй результаты в @report/tests.md
```

## 8) Промпт
```text
Подведи итоговый отчет о проделаной работе

- Создай в папке report файл report.md
- Сформируй короткий отчет в файле report.md
- Опиши какой LLM используется и как его настроить
- какое API выбрано и какие операции поддерживаются;
- как запустить агента (команды);
- укажи файл и диапазон строк, где объявлен tool и выполняется реальный вызов API (например, tools/api_tool.py:L12–L48). Также добавьте вывод результата tool в консоль для дебага (например, print()/логгер) и укажите, где именно это сделано (файл/строки).
- Убедись что агент корректно интерпретирует запросы и выбирает нужный API-метод. Подтверждение: приведите минимум 1 пример запроса, который должен привести к вызову конкретного API-метода, и укажите ожидаемый метод (например: запрос → createUser). Желательно приложить фактический фрагмент лога/trace, подтверждающий вызов tool.
- Ответ агента соответствует заданному контракту. Подтверждение: укажите, где в репозитории описаны требования к формату ответа агента (например, README.md: раздел “Контракт ответа” или prompts/system.md:L1–L30).
- Приложи 5 тестовых запросов и полученные результаты. Достаточно указать ссылку на@report/tests.md
- Создай отдельный файл prompts.md и добавь туда все промпты из этого диалога и добавь ссылку на этот файл в report.md
```
